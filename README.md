

# Gyscan-ARM

## 项目简介

Gyscan-ARM 是GYscan的arm分支版本也是一个专注于收集、整理各类安全漏洞利用代码的专业数据库项目。本项目收录了针对不同操作系统平台、Web应用程序及嵌入式设备的漏洞利用代码（Exploit），为安全研究人员、渗透测试工程师及漏洞研究人员提供全面的漏洞利用资源参考。

项目采用结构化的目录组织方式，按照目标平台（Platform）、攻击类型（Type）和文件格式进行分类存储，方便用户快速检索和定位所需的漏洞利用代码。所有收录的利用代码均来源于公开的安全研究社区，可用于合法的安全测试和研究活动。

## 主要特性

本项目具有以下核心功能特点：首先是分类体系完善，涵盖AIX、Android、ARM、ASP、ASPX、BSD、CFM、CGI等多种主流平台和应用类型；其次是攻击类型全面，包括本地提权（Local）、远程攻击（Remote）、拒绝服务（DoS）以及Web应用攻击（Webapps）等多种攻击场景；最后是格式支持广泛，支持Python、Ruby、C/C++、Perl、PHP、Shell、Bash等多种编程语言和脚本格式的漏洞利用代码。

## 使用说明

安装请安装ac密钥以保证termux可正常运行
```bash
pkg install -y ca-certificates
```

其他正常的arm架构可不用安装ac证书，除非你运行网络功能时显示tls认证失败x509，如果你是其他正常的发行版请自行Ai解决，非常抱歉～

### 环境准备

本项目的漏洞利用代码涉及多种编程语言和环境，建议准备以下运行环境：

对于Python脚本，需要Python 2.7或Python 3.x版本，部分脚本可能需要安装requests、cherrypy等依赖包；对于Ruby脚本，需要Ruby运行环境配合Metasploit框架使用；对于C/C++代码，需要GCC、Clang等编译器以及相应的构建工具；Perl脚本需要Perl解释器及相关模块。

### 检索漏洞利用代码

用户可根据目标系统和应用类型，在对应的平台目录下查找相应的漏洞利用代码。例如查找针对Android系统的本地提权利用，应进入`Client/exploitdb/exploits/android/local/`目录进行检索。文件命名通常遵循「漏洞编号.扩展名」的格式，如`35711.c`表示编号为35711的C语言漏洞利用代码。

### 代码使用注意事项

使用本项目中的漏洞利用代码时，请务必注意以下事项：仅在获得明确授权的目标系统上进行测试；详细阅读代码中的说明注释，了解其适用条件和限制；部分漏洞利用代码可能需要根据目标环境进行参数调整；在实际使用前，建议先在隔离的测试环境中验证代码功能。

## 平台覆盖

本项目目前支持的平台和攻击类型包括但不限于：

**操作系统平台**：AIX、Android、ARM、BSD（FreeBSD、OpenBSD、NetBSD）、Linux、Solaris、Windows等主流操作系统。

**Web应用技术**：ASP、ASP.NET、PHP、ColdFusion、CGI、Java Servlet等各类Web应用技术栈。

**攻击向量**：本地提权漏洞、远程代码执行、SQL注入、跨站脚本、文件上传、命令注入、认证绕过等多种漏洞类型。
